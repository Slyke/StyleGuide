Functions

Function Accent Sets

secondary accents, not status colors

Inspired by function highlight panels. These accents are condensed to three distinct secondary families plus a neutral utility set. They are for grouping and labeling, and stay outside the semantic start / mid / warning / danger state map. Accent badges stay flat as labels; accent chips keep raised chrome so they still read as controls.

States

Hover, Focus, Selected & Disabled

hover = blue, focus = amber, disabled = dashed
Tabs

Hover moves into neon blue. Selected state keeps the component in its own tone without turning the whole surface loud.

Mid and info surfaces lean neon green so they remain distinct from the amber warning color for users with green-yellow confusion.

Warning uses amber with a warm red bias, which keeps it away from the green mid tone while still reading as caution.

Error and end states sit in a strong red range so they stay visibly separate from both amber and blue.

State Legend

Hover is always a cooler blue shift. Keyboard focus is a brighter amber ring. Selected controls keep their own tone so state and meaning are both visible at the same time.

Blue = Start Green = Mid Amber = 75% Red = End
Disabled Actions

Disabled buttons use the mqttctl pattern: dashed border, low saturation, no hover lift, and no active shadow. The disabled control sets below extend that same rule to other controls.

Disabled

Disabled Control Sets

muted tone, dashed chrome, no hover or active lift

Disabled controls keep their color family, then mute it through reduced saturation, dashed borders or tracks, and no hover energy.

Feedback

Status & Data

Start: Primary actions and initial states stay in neon blue.
Mid / Info: Informational and in-progress states lean neon green.
75% / Warning: Amber remains clearly separated from green.
End / Error: Error states use a strong red tone with stronger contrast.
Neutral: Utility surfaces stay quiet and support the stronger semantic and accent families.
Accent Alpha: Secondary family A reads as grouped but does not replace a semantic status tone.
Accent Beta: Secondary family B separates from both blue and green while staying vivid.
Accent Gamma: Secondary family C stays warmer and earthier than warning amber.

Mark Row Usage Primary Token
neutral Passive controls, baseline panels, non-semantic utility state --color-control-bg
start Primary actions, hover energy, initial active states --color-start
mid In-progress information, checked state, selected state --color-mid
warning Keyboard focus, review state, caution at 75% --color-warning
danger End state, error feedback, destructive actions --color-danger
alpha Secondary accent family A for grouped controls and support surfaces --color-accent-alpha
beta Secondary accent family B for alternate grouping and differentiation --color-accent-beta
gamma Secondary accent family C for warm grouped surfaces without warning semantics --color-accent-gamma
