Style Guide / Theme Tokens

Dark UI Baseline

Flat black surfaces, no gradients, and a neon-leaning state system built around start, mid, warning, and error tones. The structure is still plain HTML and CSS, but the tokens are ready to be lifted into Svelte components later.

States

Hover, Focus & Selected

hover = blue, focus = amber
Tabs

Hover moves into neon blue. Selected state keeps the component in its own tone without turning the whole surface loud.

Mid and info surfaces lean teal so they remain distinct from the amber warning color for users with green-yellow confusion.

Warning uses amber with a warm red bias, which keeps it away from the teal mid tone while still reading as caution.

Error and end states sit in a pink-red range so they stay visibly separate from both amber and blue.

State Legend

Hover is always a cooler blue shift. Keyboard focus is a brighter amber ring. Selected controls keep their own tone so state and meaning are both visible at the same time.

Blue = Start Teal = Mid Amber = 75% Pink Red = End

Feedback

Status & Data

Start: Primary actions and initial states stay in neon blue.
Mid / Info: Informational and in-progress states lean teal-green.
75% / Warning: Amber remains clearly separated from teal.
End / Error: Error states use a pink-red tone with stronger contrast.
